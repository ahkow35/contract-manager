Decisions Outside The Spec

  • Engine = docxtemplater named tokens, not highlight-detection. Discovered the prior MY build (CoWork/Employment Contract/contract-merge.js, 26/26 tests) already proved this path; contracts need computed/validated fields highlights can't give. Old Python highlight engine kept as reference only.
  • Supabase auth is non-blocking when unconfigured. No credentials available (and no-secrets-in-chat rule), so missing env → open "dev mode" with a banner; gate turns on once NEXT_PUBLIC_SUPABASE_* are set. Lets the app run/verify locally now.
  • salary-in-words is auto-filled but operator-editable. The legal figure is load-bearing and INWW's "and" placement is house-specific; amountToWords() provides a default, operator verifies.
  • Dropped next/font/google (Geist) for a system font stack. Removes a build-time network dependency → hermetic builds.
  • NRIC M-series checksum is best-effort (high confidence S/T/F/G, moderate M); format is always gated. SG NRIC is optional in this template (signature block is hand-signed).
  • Roles stored in app_metadata, not a separate DB table. Supabase app_metadata is writable only by the service-role key (never from the browser JWT); no additional RLS complexity needed. Chose over a separate user_roles table to keep the role authoritative and unspoofable.
  • Usage dashboard records no PII beyond user ID/email + template ID + timestamp. Counts only, no document field values — satisfies INV-2 and keeps the usage_events table out of PDPA scope. RLS on usage_events: service_role only; clients cannot read each other's events.
  • Supabase Auth URL config left as a dashboard task. The CLI OAuth session token is stored chunked in macOS keychain and is rejected by the Management API. Attempting automation would require re-authenticating or injecting the service-role key differently; the dashboard path is a 30-second fix and the safer option.

Changes And Tradeoffs

  • Rebuild in place: old backend/ + frontend/ + Docker/Jenkins moved to legacy/ (git history preserved), Next.js scaffolded at root.
  • Tokenisation flattens superscript ordinals. Dates like "5th" were split across runs by superscript formatting; the span tokeniser collapses them — tokens render plain ("5th January 2026"), which matches the merge output anyway.
  • Header name/address not tokenised (ambiguous blank-for-hand-fill structure) — flagged as an open question rather than guessed, per "don't guess legal content".
  • Blank templates served from public/templates/ for the working slice (same-origin, no PII). Supabase Storage is the intended production home (INV-2).
  • next build must run with the sandbox disabled locally — Turbopack's PostCSS worker binds a port the sandbox blocks. Not a code issue; Vercel is unaffected.
